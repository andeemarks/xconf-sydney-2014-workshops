The Application

  • Back-end: A simple REST-based service written in Golang
  • Front-end: A static Swagger UI browser

Prerequisites

  • Laptop with Vagrant/Virtual Box installed
  • USB key with bootstrap Vagrant ISO and Vagrantfile

Create developer box

  1. cd $XCONF_HOME
  2. copy Vagrant ISO to $XCONF_HOME
  3. create a Vagrantfile in $XCONF_HOME: 
    
VAGRANTFILE_API_VERSION = "2"

Vagrant.configure(VAGRANTFILE_API_VERSION) do |config|
  config.vm.box = "centos-go-1.2.2"
  config.vm.box_url = "xconf-dev.box"
  config.vm.provider "virtualbox" do |vb|
    vb.customize ["modifyvm", :id, "--memory", "512"]
  end
  config.vm.network "forwarded_port", guest: 80, host: 8080
  config.vm.network "forwarded_port", guest: 8080, host: 8081
end
  4. vagrant up
  5. vagrant ssh

What's in our Box?

  • Centos 6.5
  • Golang
  • Git client
  • Apache
  • Port forwarding of 80 and 8080

Clone source repos


# Back-end service
mkdir -p $GOPATH/src/github.com/andeemarks
cd $GOPATH/src/github.com/andeemarks
git clone https://github.com/andeemarks/xconf-go-svc.git
cd xconf-go-svc
						

# Front-end service
cd ~/var/www/html
sudo git clone https://github.com/andeemarks/xconf-svc-client.git

Start back-end service


cd $GOPATH/bin
. ./xconf-go-svc

Test

  1. Browse to http://localhost:8080/xconf-svc-client/
  2. Set http://localhost:8081/apidocs.json as the URL
  3. Winning!
